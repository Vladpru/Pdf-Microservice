# PDF Microservice

Standalone FastAPI service that generates PDF exports of ideas and problems, stores them ephemerally, and serves them via short-lived signed URLs.

## How it works

1. NestJS calls `POST /generate` with an entity payload
2. Service renders an HTML template → PDF via WeasyPrint, saves to `/tmp`
3. Returns a JWT-signed download URL valid for 1 hour (configurable)
4. `GET /pdf/{token}` streams the file to the client and deletes it afterward

No database. No persistent storage.

## Local development

**Prerequisites:** Docker Desktop

```bash
cp .env.example .env
# Fill in PDF_SERVICE_SECRET and PDF_INTERNAL_SECRET in .env

docker compose up --build
```

Service is available at `http://localhost:8000`.

### Without Docker

Requires system libraries: `pango`, `cairo`, `gdk-pixbuf` (via Homebrew on macOS or apt on Linux).

```bash
pip install -r requirements.txt -r requirements-dev.txt
uvicorn app.main:app --reload
```

## Running tests

```bash
pip install -r requirements.txt -r requirements-dev.txt
pytest -v
```

> `test_pdf.py` is skipped when WeasyPrint system libraries are not available locally. All other tests run without native dependencies.

## API

### `POST /generate`

Generates a PDF and returns a signed download URL.

**Auth:** `Authorization: Bearer <PDF_INTERNAL_SECRET>`

**Request body:**
```json
{
  "entity_type": "idea",
  "entity_id": "abc-123",
  "title": "My Idea",
  "description": "Full description...",
  "ai_summary": null,
  "author": { "name": "Alice", "avatar_url": null },
  "status": "PUBLISHED",
  "stage": "PROTOTYPING",
  "tags": ["tag1"],
  "like_count": 5,
  "view_count": 100,
  "comment_count": 2,
  "published_at": "2026-01-01T00:00:00Z",
  "linked_entities": [],
  "comments": [],
  "cover_image_url": null
}
```

**Response:**
```json
{
  "download_url": "https://pdf-service.onrender.com/pdf/<token>",
  "expires_at": "2026-06-17T13:00:00Z"
}
```

### `GET /pdf/{token}`

Streams the PDF file and deletes it. Returns `401` if the token is expired or invalid, `410` if the file was already downloaded.

### `GET /health`

Returns `{"status": "ok"}`.

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `PDF_SERVICE_SECRET` | yes | — | JWT signing secret (min 32 chars) |
| `PDF_INTERNAL_SECRET` | yes | — | Bearer token for NestJS → service auth |
| `PDF_SERVICE_BASE_URL` | no | `http://localhost:8000` | Prepended to download URLs |
| `TOKEN_TTL_SECONDS` | no | `3600` | Download link lifetime in seconds |

## NestJS integration

```typescript
const response = await this.httpService.axiosRef.post(
  `${process.env.PDF_SERVICE_URL}/generate`,
  entityPayload,
  { headers: { Authorization: `Bearer ${process.env.PDF_INTERNAL_SECRET}` } }
);
// response.data = { download_url, expires_at }

this.gateway.server.to(userId).emit('pdf:ready', {
  download_url: response.data.download_url,
  expires_at: response.data.expires_at,
});
```

## Deployment (Render)

1. Connect this repo in Render and select **Docker** runtime — `render.yaml` is already configured
2. After first deploy, set these env vars manually in the Render dashboard:
   - `PDF_INTERNAL_SECRET` — same value as in the NestJS service
   - `PDF_SERVICE_BASE_URL` — public Render URL (e.g. `https://pdf-service.onrender.com`)
3. `PDF_SERVICE_SECRET` is auto-generated by Render
